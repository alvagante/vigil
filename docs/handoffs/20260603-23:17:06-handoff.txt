## Key design decisions (not in ADRs)

**Two-level expiry model (EXS-006):**
- TTL expiry makes an entry *stale* — it stays in ETS, available for stale serving.
- Janitor evicts entries only when `expires_at` is older than `hard_retention_ms` (default 10 min). This prevents unbounded ETS growth while preserving stale entries for the degraded-source path.

**Freshness tag contract:**
- `Cache.fetch/6` returns `{:ok, entry, :hit | :miss | :stale}`.
- `:stale` = expired entry served because `compute_fn` errored and a stale ETS entry existed.
- `Dispatcher.call/5` maps tags to `Result.freshness`: `:hit` → `:cached`, `:miss` → `:live` (unchanged from result), `:stale` → `:stale`.
- `Result.freshness` typespec already includes `:stale` (was pre-declared).

**ETS ownership:**
- `:vigil_cache` table is `:protected` and owned by `Cache.Server`.
- `Cache.Janitor` cannot write/delete directly — all mutations delegate to `Cache.Server` via `GenServer.call` (synchronous, important for test correctness).
