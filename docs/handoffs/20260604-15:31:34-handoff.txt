── Key context for #14 ─────────────────────────────────

1. Task discovery
   bolt task show --format json  (lists tasks with parameter metadata)
   bolt task show <task-name> --format json  (individual task detail)
   Same CLI seam as #13: FakeCLI will need a new dispatch branch for
   List.starts_with?(args, ["task"]).

2. Plan execution
   bolt plan run <plan-name> --params '{"key":"val"}' --format json
   Output shape differs from `bolt command run` — plans produce a single
   "result" object, not an "items" list. Verify against live Bolt docs.

3. RBAC for tasks/plans
   The #9 commit (af7ad54) implemented per-command RBAC on the REST API path.
   Issue #14 needs the analogous predicates for tasks and plans.
   The Dispatcher (vigil_plugin/lib/vigil/plugin/dispatcher.ex) has NO
   permission check — callers handle RBAC. The wiring point is the REST
   controller or the LiveView submit handler, depending on where #9 put it.
   Check apps/vigil_web/lib/vigil_web/controllers/api/execution_controller.ex
   and ExecutionSubmitLive for the existing per-command pattern.

4. ExecutionSubmitLive auto-forms
   Design §9 describes widget types. The existing ExecutionSubmitLive
   (apps/vigil_web/lib/vigil_web/live/execution_submit_live.ex) was wired
   for SSH command input in #7. Extend it to render parameter forms based on
   bolt task metadata.

5. Process-tree reaping (NOT in #14 scope)
   Deferred to #15 per handoff. Do not reopen in #14.
