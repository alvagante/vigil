### `telemetry.ex` fix (`apps/vigil_core/lib/vigil/telemetry.ex`)
`Mix.env()` was being called at runtime, which crashes in OTP releases.
Fixed with `@env Mix.env()` compile-time attribute.

### `runtime.exs`
`server: true` moved to top level (outside `if config_env() == :prod` guard) so the
endpoint starts when running via release in any env. Tests still pass (241, 0 failures).

### `mix.exs`
Two changes:
1. `releases/0` now lists all 9 CE apps as `:permanent` ‚Äî was missing all six integration
   apps (`vigil_auth_oidc`, `vigil_integrations_{ansible,bolt,proxmox,puppet,ssh}`).
   Without this, `Catalog.all()` returns empty in a release.
2. `"assets.deploy"` alias added: `["cmd --app vigil_web mix assets.deploy"]`.
   Makes `mix assets.deploy` work from umbrella root (required by Dockerfile).

### `Dockerfile`
Multi-stage: `hexpm/elixir:1.19.5-erlang-28.0-alpine-3.22.4` build ‚Üí `alpine:3.22` runtime.
Uses explicit per-app `COPY apps/<name>/mix.exs` lines (not glob ‚Äî Docker glob destinations
are broken for this use case). Runs as `nobody`.

### `.dockerignore`
Excludes: `.git`, `.github`, `.kirograph`, `deps`, `_build`, `*.log`, `node_modules`,
`.tool-versions`, `docs/`, `apps/*/test/`.

### `.github/workflows/ci.yml`
Three jobs: `lint` (compile + format check) ‚Üí `test` (postgres service, runs `mix test`) ‚Üí
`docker-smoke` (docker build + migrate + start + `/_health` check).
Note: `mix compile --warnings-as-errors` is NOT used in CI ‚Äî pre-existing cross-app
reference warnings in `vigil_web` referencing Bolt modules that aren't static deps
(intentional plugin architecture). The `--warnings-as-errors` flag lives only in the
`precommit` local alias.
