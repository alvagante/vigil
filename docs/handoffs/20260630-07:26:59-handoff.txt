## Key design choices
- Generator in lib/ not test/support/ — so the Mix task works in all MIX_ENV.
  Namespace Vigil.Test.Fixtures.Generator signals it's test infrastructure.
- Fixtures committed (not gitignored) — nightly CI has reproducible data without
  a generation step.
- Nightly uses `working-directory: apps/vigil_web` — umbrella root `mix test` does
  NOT recurse into apps; tests must run from the app directory.
- Deferred tests use @tag :skip (hard skip) not @tag :perf_pending, so they appear
  in the skip count in CI output rather than being silently absent.
